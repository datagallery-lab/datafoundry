# Web 工作台指南

Web 工作台是 Open Data Agent 的默认图形化入口。它适合产品试用、客户演示和日常数据分析，重点展示 Agent 如何一步步理解数据、执行查询并生成结果。

## 启动方式

在仓库根目录启动完整本地服务：

```bash
npm run dev
```

打开：

```text
http://127.0.0.1:3000/data-tasks
```

如果只启动前端，需要先启动后端：

```bash
npm run dev:api
npm run dev:web
```

前端默认连接：

```bash
NEXT_PUBLIC_AGENT_RUNTIME_URL=http://127.0.0.1:8787/api/copilotkit
```

## 界面结构

Web 工作台围绕「提问、分析、看结果、带走产出」设计，主要分为三栏：

```text
左侧：资源与任务
中间：对话与分析过程
右侧：任务控制台
```

窗口较窄时，右侧控制台可能收起为抽屉，可从聊天区顶部重新打开。

## 左侧：资源与任务

左侧负责管理工作区资源和多个数据任务。当前版本的模型选择不在左侧资源区，而是收缩到对话输入框旁的模型选择器中。

常见资源包括：


| 资源           | 用途                                                           |
| ------------ | ------------------------------------------------------------ |
| Data Sources | 查看内置演示数据、添加数据库、测试连接、浏览 schema。                               |
| Knowledge    | 管理文档集合和检索配置，适合高级分析场景。                                        |
| Agent Tools  | 管理 MCP Server 和 Skill。MCP 用于连接外部工具服务，Skill 用于影响 Agent 的分析策略。 |
| Assets       | 管理可复用工作区资产，例如后续数据任务可引用的文件。                                   |


首次体验时，只需要确认左侧的「Data Sources」和输入框旁的模型选择即可。

任务列表用于管理多轮分析：

- 新建任务：开始新的分析问题。
- 切换任务：在不同问题之间查看历史。
- 重命名：给任务起易识别的名字。
- 置顶或删除：整理常用或过期任务。

## 中间：对话与分析过程

中间区域是主要操作区。你可以直接输入自然语言问题，例如：

```text
统计 orders 表里各渠道的订单数量和 GMV 总和
```

发送后，中间区域会显示：

- Agent 的文字回复。
- 步骤卡片，例如检查 schema、执行 SQL、生成结论。
- 运行状态，例如执行中、等待输入、已完成。

点击步骤卡片时，右侧会切到详情视图，展示该步骤的输入参数和返回结果。

## 输入框能力

输入框周围包含本次分析的控制能力：


| 功能     | 用途                                   |
| ------ | ------------------------------------ |
| 模型选择   | 切换本次分析使用的 LLM。                       |
| 会话资源开关 | 控制本会话启用哪些数据源、知识库、MCP Server 和 Skill。 |
| `@` 提及 | 在单次提问中指定数据源或文件。                      |
| 停止运行   | 中断当前分析。                              |
| 文件上传   | 将文件加入本次分析上下文，部分环境可能未开放。              |


资源控制可以理解为三层：

1. 工作区默认：左侧 Data Sources、Knowledge、Agent Tools 和 Assets 配置，影响新会话默认可用资源。
2. 本会话启用：输入框底部资源开关，只影响当前任务。
3. 本次提问指定：通过 `@` 指定本轮问题使用的资源。

首次体验可以使用默认配置，不必先理解所有控制项。

## 右侧：任务控制台

右侧控制台帮助你判断一次分析是否可信、能否复盘。

### 概览

概览展示当前问题、步骤数量、成功率、产出数量、Token 用量和动态步骤列表。它适合快速判断任务进度。

### 追溯

追溯按时间顺序展示完整分析链路，例如：

1. 检查表结构。
2. 编写并执行 SQL。
3. 生成结论。

这适合回答「Agent 是如何得出结果的？」。

### 产出

产出是本次分析产生的可复用结果：


| 类型  | 说明                     |
| --- | ---------------------- |
| 表格  | 支持搜索、排序和导出 CSV。        |
| 图表  | 支持预览和导出图片，具体能力取决于当前环境。 |
| SQL | 展示 Agent 实际执行的查询。      |
| 报告  | 展示结构化分析结论。             |
| 文件  | 支持下载或在后续提问中引用。         |


### 详情

详情展示单个步骤的输入、输出、Token 用量和关联产出。没有选中步骤时，会提示你从中间区域或追溯列表选择一个步骤。

## 典型使用场景


| 场景    | 示例问题                            |
| ----- | ------------------------------- |
| 快速探表  | `这个数据源里有哪些表？每张表主要字段是什么？`        |
| 指标统计  | `按渠道统计订单量和 GMV。`                |
| 趋势分析  | `分析最近 30 天 GMV 趋势，找出波动最大的日期。`   |
| 多轮追问  | `再按品类拆分一下。`                     |
| 切换数据源 | `使用 sales-pg 数据源，统计最近 7 天订单情况。` |


## 当前边界

部分高级入口可能显示为未开放或依赖后端能力，例如复杂知识库检索、MCP 工具运行时策略、文件上传和工作区文件复用。这不影响使用内置演示数据源完成基础数据分析体验。

如需连接自己的数据源，请继续阅读 [数据源指南](data-sources.md)。
