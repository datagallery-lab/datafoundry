# v0.2.0

DataFoundry 0.2 将首个受控 Data Agent 工作台，扩展为支持并发分析、证据追问、资产复用和正式 Web 部署的有状态工作流。

## 重点更新

### 有状态分析与会话控制

- 多个数据任务会话可同时运行，各自的 live state 不互相污染。
- run 进行中可排队、编辑、立即发送或删除后续问题。
- 刷新页面或切换会话后，可恢复已持久化的消息、工具调用、run events、产出和终态。
- 可从早期用户问题重新提问，或从持久化 checkpoint 创建分支。原路径保留，并可在多个分支间显式切换。

### 证据、Trace 与语义探索

- 下一个问题可引用完整产出，或选中的表格区域和文本片段。
- Artifact、SQL audit、trace step、schema、preview 和 knowledge 引用会先在服务端解析，再进入受控 run context。
- 可通过语义 Trace DAG 查看 run、step、tool call、checkpoint、output 及它们之间的关系。
- 配置兼容的 Data Link MCP 服务后，可在工作区图中探索表、字段、概念、实体和关系边。

### 产出与工作区资产

- 会话产出可独立于实时对话恢复。
- 可预览表格、图表、报告、SQL 和文件，并导出支持的表格和图表格式。
- 文件可先上传到当前会话，再将支持的文件或文件型产出提升为跨会话工作区资产。
- 后续问题可通过 mention 引用可复用资产，从已有材料继续分析。

### Web 部署与引导

- 内置密码认证，覆盖注册、验证、登录、密码重置、会话管理、CSRF 保护和安全 Cookie。
- 正式部署中，浏览器的 API 和 CopilotKit 流量经过同源 Next.js 代理。
- Web 工作台支持英文和简体中文。
- OpenAI-compatible 模型配置可先执行连接测试，再选用到 run。
- 首次引导与自动准备的 DTC Growth Review SQLite 案例降低首次体验成本。

### 运行时与终端工作流

- 在保持受控上下文编译与 prompt snapshot 顺序稳定的同时，兼容 provider 对消息内容的约束。
- 历史恢复和继续提问仍遵守 schema-first、只读 SQL 和审计边界。
- TUI 升级为 chat-first 交互，并补齐会话恢复、产出浏览、表格、输入历史、补全、滚动和响应式终端布局。

## 升级说明

- 需要 Node.js 22 或更高版本。
- 拉取 0.2 后请重新执行 `npm install`，因为 workspace 依赖和 lockfile 已变更。
- 正式部署使用 `password` 认证与 `build` / `start`，必填认证变量见 [快速开始](../quick-start.md)。
- 修改 `NEXT_PUBLIC_*` 后需要重新构建 Web。
- Data Link 是集成界面，不是内置语义服务；需先配置兼容的 MCP Server。
- 生产部署仍需根据环境配置 Secret 管理、访问策略、审计导出、监控、备份和 TLS 终止。
