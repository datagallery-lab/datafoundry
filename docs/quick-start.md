# 快速开始

本文面向第一次使用 Open Data Agent 的用户，帮助你在本地启动产品，配置模型 API Key，并用内置演示数据源跑通一轮数据分析任务。

首次体验**不需要**准备自己的数据库。产品已内置一份演示数据，配置好模型后即可开始提问。

## 你将完成什么

完成本文后，你可以：

- 在本地启动数据分析工作台
- 配置可用的 LLM API Key
- 使用内置演示数据源发起自然语言分析
- 查看分析过程、追溯链和结果产出
- 了解工作台的主要功能区域
- （可选）连接自己的数据库继续体验

## 环境要求

- **Node.js** >= 22
- **npm**
- 一个可用的 **OpenAI-compatible LLM API Key**（例如通义千问、DeepSeek 等兼容 `/chat/completions` 的模型服务）

支持 **Linux**、**macOS** 和 **Windows**。请在同一操作系统上完成安装和运行，不要在 Windows 与 WSL 之间共用同一个 `node_modules` 目录。

## 1. 安装

在仓库根目录执行：

```bash
node -v   # 确认版本 >= 22
npm install
```

安装完成后会自动编译项目依赖，首次安装可能需要几分钟。

## 2. 配置模型 API Key

### 2.1 复制环境变量模板

```bash
cp .env.example .env
cp apps/web/.env.example apps/web/.env.local
```

### 2.2 填写根目录 `.env`

打开仓库根目录的 `.env`，至少填写以下与模型相关的变量：

```bash
LLM_PROVIDER=openai-compatible
LLM_MODEL=qwen-plus
LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
LLM_API_KEY=你的_API_Key
```

说明：


| 变量             | 作用                                         |
| -------------- | ------------------------------------------ |
| `LLM_BASE_URL` | 模型服务的 API 地址                               |
| `LLM_API_KEY`  | 模型服务的 API Key，**必填**                       |
| `LLM_MODEL`    | 默认使用的模型名称                                  |
| `LLM_PROVIDER` | 模型提供方类型，兼容 OpenAI 接口时填 `openai-compatible` |


首次体验建议直接使用根目录 `.env` 中的服务端默认模型配置。工作台里也会显示「服务端默认」这一项，无需额外在前端再配一遍。

如果你使用 DeepSeek，可参考：

```bash
LLM_PROVIDER=openai-compatible
LLM_MODEL=deepseek-chat
LLM_BASE_URL=https://api.deepseek.com
LLM_API_KEY=你的_API_Key
```

### 2.3 前端连接地址（通常无需修改）

`apps/web/.env.local` 默认如下：

```bash
NEXT_PUBLIC_AGENT_RUNTIME_URL=http://127.0.0.1:8787/api/copilotkit
```

本地默认启动时一般不需要改动。只有当你修改了后端端口或部署地址时，才需要同步更新这里。

## 3. 启动产品

在仓库根目录执行：

```bash
npm run dev
```

启动成功后：

- 前端工作台：[http://127.0.0.1:3000/data-tasks](http://127.0.0.1:3000/data-tasks)
- 后端健康检查：[http://127.0.0.1:8787/healthz](http://127.0.0.1:8787/healthz)

浏览器打开 `/data-tasks` 即可进入**数据分析工作台**。

## 4. 工作台能帮你做什么

数据分析工作台是 Open Data Agent 的主界面。你可以把它理解成「带 AI 助手的数据分析 IDE」：

- **用自然语言提问**，不必手写 SQL
- **自动检查表结构**，Agent 会先确认字段再查询，减少瞎猜
- **只读查询**，适合探索性分析，不会修改你的数据
- **完整追溯**，每一步检查、查询、推理都能在右侧看到
- **结果可带走**，表格可导出 CSV，图表可导出 PNG
- **多任务并行管理**，不同问题可以开不同会话分别跟进

首次体验只需要：**配好模型 → 打开工作台 → 直接提问**。内置演示数据已经准备好，无需额外导入。

## 5. 使用内置演示数据源

产品已内置一份 **DuckDB 演示数据源**，其中包含一张示例表 `orders`，字段包括：

- `order_id`：订单 ID
- `channel`：渠道
- `category`：品类
- `user_type`：用户类型
- `gmv`：成交金额
- `created_at`：创建日期

首次进入工作台时，左侧「数据源」区域会默认启用这份演示数据。你**不需要**先配置数据库，也**不需要**导入任何文件。

## 6. 跑通第一个分析任务

### 6.1 新建数据任务

1. 打开 [http://127.0.0.1:3000/data-tasks](http://127.0.0.1:3000/data-tasks)
2. 在左侧点击「新建数据任务」
3. 确认当前使用的是默认演示数据源
4. 在底部输入框确认模型已选中（默认是「服务端默认」）

### 6.2 发送第一个问题

你可以直接输入自然语言，也可以点击欢迎页上的示例卡片。推荐从下面任一问题开始：

**查看表结构：**

```text
帮我查看数据源里有哪些表，并说明每张表的主要字段
```

**简单统计：**

```text
统计 orders 表里各渠道的订单数量和 GMV 总和
```

**趋势分析：**

```text
分析 orders 表按日期的 GMV 变化，找出波动最大的日期
```

### 6.3 观察结果

发送问题后，你会看到三部分内容同步更新：

1. **中间对话区**：Agent 的回复，以及「检查表结构」「执行 SQL」等步骤卡片
2. **右侧控制台 · 概览**：当前问题、步骤数、成功率、产出数量
3. **右侧控制台 · 追溯 / 产出**：完整分析链路和查询结果

如果一切正常，说明你已经成功跑通第一个数据分析任务。

## 7. 工作台功能介绍（用户视角）

打开工作台后，你会看到一个围绕「提问 → 分析 → 看结果 → 带走产出」设计的三栏界面。下面按**你会看到什么、能做什么**来介绍。

### 7.1 界面总览

```text
┌─────────────────┬────────────────────────────┬─────────────────┐
│  左侧           │  中间                      │  右侧           │
│  资源与会话     │  对话与分析过程            │  任务控制台     │
│                 │                            │                 │
│  · 数据源       │  · 欢迎页 / 对话历史       │  · 概览         │
│  · 模型         │  · 步骤卡片                │  · 追溯         │
│  · 知识库等     │  · 输入框与模型选择        │  · 产出         │
│  · 会话列表     │  · 运行状态                │  · 详情         │
└─────────────────┴────────────────────────────┴─────────────────┘
```

**推荐使用方式：**

1. 在左侧确认数据源和模型
2. 在中间用自然语言提问
3. 在右侧查看分析链路和查询结果

窗口较窄时，右侧控制台会收起到抽屉里，点击聊天区顶部的「打开控制台」即可展开。

### 7.2 左侧：资源管理与会话

左侧是你的「工作区面板」，负责管理**长期资源**和**多个分析任务**。

#### 工作区资源入口

点击左侧的配置行，可以打开对应的管理面板：


| 入口        | 你可以做什么                           |
| --------- | -------------------------------- |
| **数据源**   | 查看内置演示数据、添加自己的数据库、测试连接、浏览表结构     |
| **模型**    | 管理 LLM 配置，测试 API Key 是否可用，切换不同模型 |
| **知识库**   | 上传文档、管理索引（高级功能，部分环境可能尚未开放）       |
| **MCP**   | 连接外部工具服务（高级功能）                   |
| **Skill** | 管理分析技能包，影响 Agent 的分析策略           |


首次体验时，你只需要关注**数据源**和**模型**。其余入口可以之后再探索。

#### 会话列表（数据任务）

每个「数据任务」对应一次独立的分析对话，类似 ChatGPT 里的一个会话：

- **新建数据任务**：开始一个全新的分析问题
- **切换会话**：在不同问题之间来回查看，互不干扰
- **搜索会话**：快速找到历史任务
- **重命名**：给任务起一个容易识别的名字（例如「5 月渠道分析」）
- **置顶**：把常用任务固定在列表顶部
- **删除**：清理不再需要的任务

会话标题会在你发送第一个问题后自动更新；你也可以随时手动改名。

#### 工作区文件

如果你上传过 CSV、Excel 等文件，或把某次分析结果加入工作区，可以在这里统一管理，并在后续提问时用 `@` 引用。

### 7.3 中间：对话与分析过程

中间是主操作区，大部分时间你会在这里提问和观察 Agent 工作。

#### 欢迎页

新建任务且还没有消息时，会看到欢迎页，内容包括：

- 产品简介：用自然语言提问，Agent 会自动检查表结构、执行只读 SQL
- 能力标签：SQL 查询、Schema 检查、数据追溯
- **示例问题卡片**：点击即可填入输入框，适合第一次体验

#### 对话与步骤卡片

发送问题后，中间区域会显示：

- **Agent 的文字回复**：解释分析思路和结论
- **步骤卡片**：例如「检查数据源 Schema」「执行只读 SQL」，让你知道 Agent 正在做什么
- **运行状态**：顶部会显示「执行中」「等待输入」「已完成」等状态

你可以点击步骤卡片，右侧会自动切到「详情」标签页，查看该步骤的参数和结果。

#### 输入框周边功能

输入框不只是打字的地方，还集成了多种控制项：


| 功能         | 位置     | 作用                     |
| ---------- | ------ | ---------------------- |
| **模型选择**   | 输入框旁   | 切换本次分析使用的 LLM          |
| **会话资源开关** | 输入框底部  | 控制本会话启用哪些数据源、知识库等      |
| `**@` 提及** | 输入框上方  | 单次提问指定要用的数据源或文件        |
| **停止运行**   | 运行中可见  | 中断当前分析                 |
| **上传文件**   | `+` 菜单 | 附加 CSV、图片等（部分环境可能尚未开放） |


#### 三层资源控制（怎么理解「默认 / 本会话 / 本次提问」）

工作台对数据源等资源有三层控制，从粗到细：

1. **工作区默认（左侧配置）**
  你在左侧添加的数据源、模型等，默认对所有新会话可用。
2. **本会话启用（输入框底部 pill）**
  某个会话里，你可以临时关闭某个数据源，不影响其他会话。例如这个任务只想查演示数据，就把其他库关掉。
3. **本次提问指定（`@` 提及）**
  某一次提问只想用特定数据源或文件时，在输入框里 `@` 它即可。发送后 `@` 会自动清空。

日常使用时，**第一次体验可以全部用默认设置**，不必理解这三层细节；需要精细控制时再回来查阅。

#### 协作式交互

分析过程中，Agent 有时会主动向你确认：

- **选择题**：例如「你想看哪个时间范围？」——直接点选项回答
- **补充说明**：需要你输入更多背景信息
- **计划审批**：Agent 提出分析计划，你可以选择批准或拒绝后再继续

这类交互会出现在对话流中，状态显示为「等待输入」或「协作中」。

### 7.4 右侧：任务控制台

右侧控制台帮你从**全局视角**理解一次分析任务，而不只是看聊天回复。

#### 概览

适合快速判断「这次分析进行得怎么样」：

- 当前问题是什么
- 一共执行了多少步骤、成功率如何
- 产生了多少份结果
- Token 用量和成本（如果模型服务已上报）

还会列出动态步骤清单，点击任一步骤可跳到详情。

#### 追溯

按时间顺序展示完整分析链路，例如：

1. 检查表结构
2. 编写并执行 SQL
3. 生成结论

适合回答「Agent 是怎么一步步得出结果的？」。支持**放大全屏**，方便截图或复盘。

#### 产出

这里存放本次分析产生的**可带走的结果**，常见类型包括：


| 类型       | 你能做什么                   |
| -------- | ----------------------- |
| **数据表格** | 搜索、排序、导出 CSV            |
| **图表**   | 切换柱状图 / 折线图 / 饼图，导出 PNG |
| **SQL**  | 查看 Agent 实际执行的查询语句      |
| **报告**   | 阅读结构化分析结论               |
| **文件**   | 下载、引用到后续提问              |


点击产出卡片会在当前标签页内展开预览，不必来回切换。

#### 详情

当你点击某个步骤卡片或追溯条目时，详情页会显示：

- 该步骤做了什么（检查 schema、跑 SQL 等）
- 输入参数和返回结果
- 单步 Token 用量
- 这一步产生了哪些产出，以及产出之间的关联

没有选中步骤时，详情页会提示你从中间或追溯里点选一个步骤。

### 7.5 典型使用场景


| 场景         | 你可以这样用                         |
| ---------- | ------------------------------ |
| **快速探表**   | 问「有哪些表？每张表主要字段是什么？」            |
| **指标统计**   | 问「按渠道统计订单量和 GMV」               |
| **趋势分析**   | 问「最近 30 天 GMV 趋势，找出波动最大的日期」    |
| **多轮追问**   | 在同一任务里继续问「再按品类拆一下」「导出成表格」      |
| **切换数据源**  | 接入自己的库后，新建任务并在底部 pill 里启用对应数据源 |
| **对比不同模型** | 同一问题分别用两个模型跑，对比结论和速度           |


### 7.6 当前版本的功能边界

工作台会诚实标注尚未开放的能力，显示为「后端未支持」。以下情况属于正常表现，不影响基础数据分析体验：

- 知识库检索、MCP 工具调用、Skill 运行时策略
- 聊天中上传数据文件或图片
- 把会话产出一键加入工作区文件库

使用内置演示数据源 + 配置好模型 API Key，即可完成本文描述的全部核心体验。

## 8. 使用自己的数据库（可选）

跑通内置演示数据后，你可以连接自己的数据库继续体验。

### 8.1 打开数据源配置

1. 在左侧工作区点击「数据源」
2. 点击「新增数据源」
3. 填写连接信息

当前支持的数据源类型包括：

- DuckDB（内置 demo）
- SQLite（本地文件）
- CSV / Excel（本地文件）
- PostgreSQL / MySQL（服务端数据库）

建议首次接入时使用**只读账号**或**本地测试库**，避免对生产数据造成风险。

### 8.2 测试连接并查看表结构

1. 保存数据源后，点击「测试连接」
2. 执行 schema 检查，确认表结构已被识别
3. 在 Schema 浏览器中查看可用表和字段

### 8.3 切换到自有数据源提问

1. 新建或切换到一个数据任务
2. 在输入框底部确认当前启用的数据源包含你刚添加的库
3. 发送与你的业务表相关的问题，例如：

```text
先查看这个数据源有哪些表，然后帮我统计最近 7 天的核心指标
```

## 9. 在前端单独配置模型（可选）

除了根目录 `.env` 中的服务端默认模型，你也可以在工作台里新增独立的模型配置：

1. 在左侧打开「模型」
2. 点击「新增模型」
3. 填写 Provider、Base URL、API Key、模型名称
4. 点击「测试连接」确认可用
5. 在聊天输入框旁的模型选择器中切换到新模型

适合以下场景：

- 同一台机器上想切换不同模型供应商
- 不同任务需要使用不同模型
- 不想修改根目录 `.env`

## 10. 常见问题

### 启动后页面打不开

- 确认 Node.js 版本 >= 22
- 确认 `npm install` 已在当前操作系统下完整执行
- 确认 3000 端口未被其他程序占用

### 发送问题后没有响应

- 检查根目录 `.env` 中的 `LLM_API_KEY` 是否已填写
- 检查 `LLM_BASE_URL` 和 `LLM_MODEL` 是否与你的模型服务匹配
- 访问 [http://127.0.0.1:8787/healthz](http://127.0.0.1:8787/healthz)，确认后端已正常启动

### 前端提示无法连接服务

- 确认 `npm run dev` 已成功启动后端
- 检查 `apps/web/.env.local` 中的 `NEXT_PUBLIC_AGENT_RUNTIME_URL` 是否指向正确地址

### 某些功能显示「后端未支持」

工作台会按当前版本能力显示可用功能。部分高级能力（如知识库运行时检索、MCP 运行时挂载、文件上传等）可能尚未对你的环境开放，这不影响使用内置演示数据源完成基础体验。

### 接入自己的数据库失败

- 确认数据库地址、端口、账号和密码正确
- PostgreSQL / MySQL 需确保网络可达
- SQLite / CSV / Excel 需确认本地文件路径有效
- 建议使用只读账号进行首次测试
